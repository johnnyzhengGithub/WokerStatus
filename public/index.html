Products / Modules

把团队状态当作一个「产品面板」实时展示。

Live Presence

  • 每个人的当前状态：Working / Waiting / Blocked / Idle
  • 任务标题、原因、等待对象一目了然
  • 状态更新即刻通过 WebSocket 广播
See live wall →

Pipeline Handoff

  • 按交付链阶段组织任务与所有权
  • 看清从构想到上线的每一次移交
  • 发现「卡在中间」的责任空档

Waiting Graph

  • 可视化「谁在等谁」的依赖关系
  • 找到最长的等待链条与瓶颈节点
  • 帮助 PM 主动打断无效等待

SLA Heat

  • WAITING 超过 30 分钟自动变黄
  • > 2 小时自动变红，优先处理
  • 一眼看到哪张卡已经「烧起来」

Audit Trail

  • 记录谁在什么时候改了什么状态
  • 追踪「为什么今天特别慢」的根因
  • 帮助回顾决策与响应的时机

Alerts

  • 阻塞 / 超时自动推送到你常用的工具
  • 减少「事后才知道」的翻车惊喜
  • 支持团队自定义阈值与通知通道

How it works

本地起一个小服务，2 分钟内看到团队状态墙。

  1. 在本地启动 WorkersStatus 服务。
  2. 发一个 POST /api/presence/update 更新自己的状态。
  3. 打开 Dashboard，实时看到 6 人状态墙变化。

Start monitoring in 2 minutes.
